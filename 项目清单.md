# 锐竞电竞点单系统 - 项目清单

## 一、项目概述

锐竞电竞点单系统是一个前后端分离的电竞服务交易平台，为用户提供游戏陪玩、代练等服务的在线交易功能。系统包含用户端、服务商端和管理端三个角色，支持服务发布、订单管理、在线支付、即时通讯、评价系统等核心功能。

### 项目名称
- **中文名**: 锐竞电竞点单系统
- **英文名**: Dianjing Esports Ordering Platform
- **项目标识**: dianjing-platform

## 二、技术栈

### 后端技术栈
- **框架**: Spring Boot 3.2.5
- **开发语言**: Java 17
- **数据库**: SQLite (JDBC驱动: sqlite-jdbc)
- **ORM框架**: Spring Data JPA + Hibernate
- **安全认证**: Spring Security + JWT (jjwt 0.12.5)
- **API文档**: SpringDoc OpenAPI + Swagger UI
- **缓存**: Redis
- **实时通讯**: WebSocket
- **构建工具**: Maven
- **工具库**: Lombok 1.18.30

### 前端技术栈
- **框架**: Vue 3.5.32
- **开发语言**: TypeScript 6.0.2
- **UI组件库**: Element Plus 2.13.7
- **路由**: Vue Router 4.6.4
- **状态管理**: Pinia 3.0.4
- **HTTP客户端**: Axios 1.15.0
- **图表库**: ECharts 6.0.0 + vue-echarts 8.0.1
- **WebSocket**: SockJS Client 1.6.1 + STOMP.js 7.3.0
- **构建工具**: Vite 8.0.4
- **样式预处理**: Sass 1.99.0

### 4.7.1 密码加密

系统使用 **BCrypt** 算法进行密码加密，通过 Spring Security 提供的 `BCryptPasswordEncoder` 实现：

**配置位置**: `backend/src/main/java/com/dianjing/config/SecurityConfig.java:67-70`

```java
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}
```

**BCrypt 特点**:
- 自动加盐处理，防止彩虹表攻击
- 自适应单向哈希算法
- 可配置工作因子（cost factor）调整计算强度
- 每次生成的哈希值都不同，但验证时能正确匹配

**密码强度验证**:
系统还提供了密码强度评估工具 `PasswordUtil`：
- 检查密码长度（6/8/12位三个等级）
- 检查字符类型（小写、大写、数字、特殊字符）
- 强度等级：非常弱 → 弱 → 一般 → 较强 → 强 → 非常强
- 密码验证要求：至少6位，至少包含2种字符类型

## 六、数据库设计

### 6.1 数据库配置
- **数据库类型**: SQLite
- **数据库文件**: `${user.dir}/game.db` (运行时自动创建)
- **ORM**: Hibernate (ddl-auto: update - 自动更新表结构)

### 6.2 数据表概览

系统共包含 **18张数据表**，详见4.1节实体类说明。

### 6.3 核心表关系

```
user (用户表)
  ├── 1:N → service (服务表) - 作为服务商发布的服务
  ├── 1:N → orders (订单表) - 作为用户的订单
  ├── 1:N → orders (订单表) - 作为服务商的订单
  ├── 1:N → review (评价表) - 作为评价者
  ├── 1:N → review (评价表) - 作为被评价者
  ├── 1:N → game_account (游戏账号表)
  ├── 1:N → service_favorite (服务收藏表)
  ├── 1:N → payment_record (支付记录表)
  ├── 1:N → withdrawal_application (提现申请表)
  ├── 1:N → complaints (投诉表) - 作为投诉人
  ├── 1:N → complaints (投诉表) - 作为被投诉人
  ├── 1:N → conversation (会话表) - user1
  ├── 1:N → conversation (会话表) - user2
  └── 1:N → message (消息表)

service (服务表)
  ├── N:1 → user (用户表) - 服务商
  ├── 1:N → orders (订单表)
  ├── 1:N → review (评价表)
  ├── 1:N → service_favorite (服务收藏表)
  └── N:M → service_tag (服务标签表) - 通过 service_tag_relation

orders (订单表)
  ├── N:1 → user (用户表) - 下单用户
  ├── N:1 → user (用户表) - 服务商
  ├── N:1 → service (服务表)
  ├── N:1 → game_account (游戏账号表)
  ├── N:1 → activity (活动表)
  ├── 1:1 → review (评价表)
  └── 1:N → payment_record (支付记录表)

user_level (等级表)
  └── 1:N → user (用户表)

activity (活动表)
  └── N:M → service (服务表) - 通过 activity_service

conversation (会话表)
  ├── N:1 → user (用户表) - user1
  ├── N:1 → user (用户表) - user2
  └── 1:N → message (消息表)
```

    redis:
      host: localhost           # Redis地址

  servlet:
    multipart:
      max-file-size: 10MB       # 最大文件上传大小

jwt:
  secret: dianjing-platform-secret-key-that-is-at-least-256-bits-long-for-security
  expiration: 86400000          # Token过期时间(毫秒)

springdoc:
  api-docs:

    path: /swagger-ui.html

file:
  upload-dir: uploads/          # 文件上传目录

## 十四、总结

锐竞电竞点单系统是一个功能完善的电竞服务交易平台，采用前后端分离架构，具有以下特点：

1. **技术栈先进**: 使用 Spring Boot 3 + Vue 3 + TypeScript 等主流技术
2. **功能完整**: 涵盖用户、服务、订单、支付、评价、即时通讯等核心模块
3. **角色分明**: 普通用户、服务商、管理员三种角色，权限清晰
4. **安全可靠**:
   - JWT认证
   - Spring Security安全框架
   - **BCrypt密码加密**（自动加盐，防止彩虹表攻击）
   - 密码强度验证
   - Token黑名单机制
   - 限流过滤器
5. **性能优化**:
   - **完善的Redis缓存策略**，包含5个缓存服务：
     - 用户缓存（信息、余额）
     - 服务缓存（详情、推荐）
     - 统计缓存（平台统计、排行榜、服务商统计）
     - 系统设置缓存
     - Token黑名单缓存
   - 缓存Key规范化命名
   - 合理的TTL设置
   - Jackson JSON序列化
   - 数据库索引优化
6. **用户体验**: Element Plus UI组件库、WebSocket实时通讯、响应式设计

该项目可作为电竞服务平台的基础框架，根据实际需求进行进一步扩展和定制。
