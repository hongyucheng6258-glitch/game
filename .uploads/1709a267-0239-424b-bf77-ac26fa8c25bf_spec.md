# 电竞陪玩代打点单平台规范文档

## 1. 项目概述

### 1.1 项目背景

随着电竞行业的快速发展，电竞陪玩和代打服务需求日益增长。本项目旨在构建一个专业、安全、高效的电竞陪玩代打点单平台，连接需求方（玩家）和服务方（陪玩/代打），提供便捷的交易服务。

### 1.2 项目目标

- 构建一个功能完整、用户体验良好的电竞陪玩代打平台
- 实现用户注册、登录、认证等基础功能
- 实现陪玩/代打服务发布、搜索、下单、支付等核心功能
- 实现订单管理、评价系统、消息通知等辅助功能
- 确保系统安全、稳定、高性能

## 2. 技术栈

### 2.1 后端技术栈

- **框架**: Spring Boot 3.2.x
- **数据库**: SQLite (文件路径: backend/game.db)
- **缓存**: Redis (无密码)
- **开发语言**: Java 17
- **构建工具**: Maven
- **API文档**: SpringDoc OpenAPI 3
- **认证授权**: JWT

### 2.2 前端技术栈

- **框架**: Vue 3
- **构建工具**: Vite
- **状态管理**: Pinia
- **路由**: Vue Router
- **UI组件库**: Element Plus
- **HTTP客户端**: Axios
- **开发语言**: TypeScript

## 4. 功能模块

### 4.1 用户管理

- **注册登录**: 手机号/邮箱注册，密码登录，验证码登录，第三方登录（微信、QQ、微博）
- **身份认证**: 实名认证，游戏账号认证，陪玩/代打资质认证
- **用户信息**: 个人资料编辑，头像上传，封面图设置，个人简介
- **角色管理**: 普通用户，陪玩/代打，管理员
- **账户安全**: 密码修改，手机/邮箱绑定与解绑，登录设备管理
- **第三方账号绑定**: 微信、QQ、微博账号绑定与解绑

### 4.2 服务管理

- **服务发布**: 陪玩/代打发布服务，设置价格、游戏类型、服务描述、服务时间段、擅长位置/英雄、服务标签等
- **服务编辑**: 修改已发布服务的信息
- **服务上下架**: 服务者可以随时上下架自己的服务
- **服务搜索**: 按游戏类型、服务类型、价格、评分、服务者性别、擅长位置等条件搜索
- **服务筛选**: 支持按多种条件组合筛选服务
- **服务排序**: 支持按价格、评分、销量、发布时间等排序
- **服务详情**: 查看服务详情，包括服务者信息、评价、历史订单、服务者游戏账号信息等
- **服务收藏**: 收藏感兴趣的服务，方便后续查看
- **服务分享**: 支持分享服务到第三方平台
- **服务预约**: 支持预约服务者的特定时间段
- **服务者排行榜**: 按评分、销量、人气等排名的服务者榜单

### 4.3 订单管理

- **下单流程**: 选择服务，填写需求，确认订单，支付，服务开始，服务结束，评价
- **订单状态**: 待支付，待服务，服务中，待评价，已完成，已取消，退款中，已退款
- **订单列表**: 按状态、时间等筛选查看历史订单，支持分页
- **订单详情**: 查看订单完整信息，包括服务信息、服务者信息、支付信息、服务要求等
- **订单操作**: 取消订单，确认完成，申请退款，评价服务，延长服务时长
- **订单提醒**: 订单状态变更提醒，服务开始提醒，支付超时提醒
- **订单统计**: 按时间、服务类型等维度统计订单数据
- **订单导出**: 支持导出订单数据为Excel格式
- **服务计时**: 陪玩服务开始后的实时计时功能
- **订单纠纷**: 订单纠纷处理机制，平台介入调解

### 4.4 支付系统

- **支付方式**: 微信支付，支付宝支付，余额支付，银行卡支付
- **支付流程**: 生成订单，发起支付，支付回调，订单状态更新，资金结算
- **余额管理**: 账户余额查询，充值，提现，提现申请审核
- **充值方式**: 微信充值，支付宝充值，银行卡充值
- **提现规则**: 提现手续费，提现限额，提现到账时间
- **支付记录**: 查看所有支付、充值、提现记录
- **发票管理**: 申请发票，查看发票状态，下载电子发票
- **资金安全**: 支付密码设置，交易密码验证，资金流水加密

### 4.5 评价系统

- **服务评价**: 对完成的服务进行评价，包括评分（1-5星）、文字评价、评价标签（如：技术好、态度好、响应快等）
- **追加评价**: 支持在首次评价后追加评价
- **评价展示**: 服务者的评价列表，评分统计，评价标签云
- **评价回复**: 服务者可以回复评价，回复后通知评价者
- **评价审核**: 平台对评价内容进行审核，过滤违规内容
- **评价举报**: 用户可以举报违规评价
- **匿名评价**: 支持匿名评价功能
- **评价筛选**: 支持按评分、评价标签、评价时间等筛选评价
- **评价统计**: 服务者的评分分布，评价关键词分析

### 4.6 消息通知

- **系统消息**: 订单状态变更，活动通知，审核结果通知，系统公告等
- **聊天功能**: 用户与服务者之间的实时聊天，支持文字、表情、图片发送
- **消息推送**: 重要消息推送提醒，支持移动端推送
- **消息列表**: 按消息类型、未读状态等筛选查看消息
- **消息已读**: 标记消息为已读，一键标记全部已读
- **消息删除**: 删除消息，清空消息
- **聊天记录**: 保存聊天记录，支持查看历史聊天记录
- **消息免打扰**: 支持设置消息免打扰时间段
- **@提及功能**: 聊天中支持@特定用户
- **消息撤回**: 支持撤回2分钟内发送的消息

### 4.7 后台管理

- **用户管理**: 查看、编辑、禁用、删除用户账号，用户身份认证审核，用户角色管理，用户批量操作
- **服务管理**: 审核、编辑、下架、删除服务，服务批量操作，服务分类管理，服务标签管理
- **订单管理**: 查看所有订单，处理订单纠纷，订单状态手动更新，订单数据导出，订单批量操作
- **数据统计**: 平台运营数据统计，包括订单量、交易额、用户增长、服务者增长等，数据可视化图表，数据导出
- **财务管理**: 查看平台收支明细，处理提现申请，平台手续费设置，财务报表生成
- **内容管理**: 审核评价内容，处理评价举报，管理系统公告，管理活动内容
- **系统设置**: 平台基本信息设置，支付方式配置，短信服务配置，推送服务配置，权限管理，日志管理
- **数据分析**: 用户行为分析，服务热度分析，订单转化分析，运营效果分析
- **安全管理**: 登录日志查看，操作日志查看，异常行为监测，安全告警设置
- **API管理**: API接口配置，API密钥管理，接口调用统计

## 6. API设计

### 6.1 API基础信息

- **API前缀**: /api/v1
- **认证方式**: JWT Token（在请求头中携带 Authorization: Bearer {token}）

## 8. 部署方案

### 8.1 开发环境

- **JDK**: 17
- **Maven**: 3.8.x
- **Node.js**: 16.x
- **Redis**: 6.x
- **SQLite**: 3.x

### 8.2 生产环境

- **服务器**: Linux CentOS 7.x

- **Nginx**: 1.20.x（用于前端静态资源部署和反向代理）
- **Redis**: 6.x
- **SQLite**: 3.x
- **PM2**: 用于管理Node.js进程（可选）

### 8.3 部署步骤

#### 8.3.1 后端部署

1. 打包Spring Boot应用：`mvn clean package -DskipTests`
2. 将生成的jar文件上传到服务器
3. 创建application.properties配置文件，配置数据库连接、Redis连接等
4. 使用nohup命令启动应用：`nohup java -jar backend.jar > backend.log 2>&1 &`

#### 8.3.2 前端部署

1. 构建Vue应用：`npm run build`
2. 将生成的dist目录下的文件上传到服务器
3. 配置Nginx，将前端请求代理到前端静态资源，将API请求代理到后端服务

#### 8.3.3 Redis部署

1. 安装Redis：`yum install redis`
2. 启动Redis服务：`systemctl start redis`
3. 设置Redis开机自启：`systemctl enable redis`
4. 配置Redis，确保无密码访问（根据需求）

## 9. 开发规范

### 9.1 后端开发规范

#### 9.1.1 代码结构

- 遵循Spring Boot最佳实践，采用分层架构
- controller层：处理HTTP请求，返回响应
- service层：实现业务逻辑
- repository层：数据访问层，与数据库交互
- model层：实体类，与数据库表对应
- dto层：数据传输对象，用于前后端数据交互
- util层：工具类
- config层：配置类

#### 9.1.2 命名规范

- 类名：采用大驼峰命名法，如：UserController
- 方法名：采用小驼峰命名法，如：getUserInfo
- 变量名：采用小驼峰命名法，如：userId
- 常量名：全部大写，单词间用下划线分隔，如：MAX\_PAGE\_SIZE
- 包名：全部小写，如：com.example.dianjing.user

#### 9.1.3 代码风格

- 使用4个空格缩进
- 每行代码长度不超过120个字符
- 方法长度不超过100行
- 类长度不超过1000行
- 适当添加注释，解释复杂业务逻辑

### 9.2 前端开发规范

#### 9.2.1 代码结构

- 遵循Vue 3最佳实践，采用组合式API
- components目录：存放通用组件和业务组件
- views目录：存放页面组件
- router目录：路由配置
- store目录：状态管理
- api目录：API请求封装
- utils目录：工具类
- assets目录：静态资源（图片、样式等）

#### 9.2.2 命名规范

- 组件名：采用大驼峰命名法，如：ServiceCard
- 方法名：采用小驼峰命名法，如：getServiceList
- 变量名：采用小驼峰命名法，如：serviceId
- 常量名：全部大写，单词间用下划线分隔，如：MAX\_PAGE\_SIZE
- 文件命名：采用短横线分隔命名法，如：service-list.vue

#### 9.2.3 代码风格

- 使用2个空格缩进
- 每行代码长度不超过120个字符
- 组件逻辑清晰，职责单一
- 适当添加注释，解释复杂业务逻辑

## 10. 安全规范

### 10.1 后端安全

- 使用JWT进行身份认证，设置合理的过期时间
- 密码加密存储（使用BCrypt等强哈希算法）
- API请求参数校验，防止SQL注入、XSS攻击等
- 实现接口限流，防止恶意请求
- 敏感数据加密传输（使用HTTPS）
- 定期备份数据库

### 10.2 前端安全

- 防止XSS攻击，对用户输入进行过滤和转义
- 防止CSRF攻击，使用Token验证
- 敏感数据不在前端存储
- 使用HTTPS协议
- 定期更新依赖库，修复安全漏洞

- 资源压缩和合并
